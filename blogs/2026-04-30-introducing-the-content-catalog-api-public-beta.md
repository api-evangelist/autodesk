            <div class="clearfix text-formatted field field--name-field-body field--type-text-with-summary field--label-hidden field__item"><pre>
&nbsp;</pre><p>We’re excited to announce the <strong>public beta release of the Content Catalog API</strong> on Autodesk Platform Services (APS)!</p>
<p>This initial release enables developers to <strong>programmatically discover and download BIM content</strong> from Content Catalog—unlocking new ways to integrate high-quality design assets directly into your applications and workflows.</p>
<hr />
<h2>What is Content Catalog?</h2>
<p>Content Catalog is a <strong>cloud-based system for organizing and distributing BIM content</strong>, such as Revit families and other design assets, across teams and projects.</p>
<p>It helps organizations ensure that designers are always using <strong>trusted, up-to-date content</strong>, improving consistency and efficiency throughout the design process.</p>
<p>With the new API, you can now bring this content into your own applications—programmatically.</p>
<hr />
<h2>What’s available in this public beta?</h2>
<p>The current release focuses on <strong>content access and retrieval</strong>. With the Content Catalog API, you can:</p>
<ul>
<li>Browse available content libraries and categories</li>
<li>Retrieve metadata for catalog items</li>
<li><strong>Download content files for use in your workflows</strong></li>
</ul>
<p>In short:&nbsp;<strong>Find the right content, and get it where you need it—via API.</strong></p>
<hr />
<h2>What can you build today?</h2>
<p>Even with this initial scope, there are powerful use cases:</p>
<ul>
<li>
<h3>Embed content into your apps</h3>
</li>
</ul>
<p>Bring Content Catalog directly into your internal tools, portals, or design assistants.</p>
<ul>
<li>
<h3>Streamline content access</h3>
</li>
</ul>
<p>Enable teams to quickly find and download approved BIM content without leaving your workflow.</p>
<ul>
<li>
<h3>Power automation pipelines</h3>
</li>
</ul>
<p>Integrate content retrieval into automated processes (e.g., design generation, validation, or setup scripts).</p>
<ul>
<li>
<h3>Enhance developer experiences</h3>
</li>
</ul>
<p>Build custom UIs or services that surface relevant BIM content based on context or user needs.</p>
<ul>
<li>

</li>
</ul>
<h2>
</h2>
<hr />
<h2>Why this matters</h2>
<p>Access to high-quality BIM content is critical—but often fragmented across systems.</p>
<p>The Content Catalog API helps address this by:</p>
<ul>
<li>Providing a <strong>centralized, API-accessible source of truth</strong></li>
<li>Reducing friction in finding and using the right assets</li>
<li>Enabling <strong>seamless integration into modern workflows</strong></li>
</ul>
<hr />
<h2>What’s coming next?</h2>
<p>This public beta is just the beginning.</p>
<p>We’re actively working on expanding the API to support more advanced capabilities, including:</p>
<ul>
<li>Content and library management</li>
<li>Metadata updates and lifecycle workflows</li>
<li>Deeper integrations across the APS ecosystem</li>
</ul>
<p>Stay tuned—there’s much more ahead.</p>
